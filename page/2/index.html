蓝桥杯练习（二）

今天接着前几天的蓝桥杯练习，把剩的几道题做完。

  1. 扑克序列
    AA223344， 一共4对扑克牌。请你把它们排成一行。
    要求：两个A中间有1张牌，两个2之间有2张牌，两个3之间有3张牌，两个4之间有4张牌。如
    4A3A2432
    2342A3A4
    请填写出所有符合要求的排列中，字典序最小的那个。
    例如：22AA3344 比 A2A23344 字典序小。当然，它们都不是满足要求的答案。
           解：因为符合条件的解的情况很少，所以先把解求出来再判断就好

    public class B_2014_7 {
    public static int count = 0; 
    public static void main(String [] args){
        char [] num = new char[8];
        f(num,0);
    }

    public static void f(char []num,int i){
        if(i >= num.length){
            if(judge(num)){
                show(num);
                count++;
            }
        }else{
            for(int j=1;j<=4;j++){
                num[i] = (char)(j + '0');
                f(num,i+1);
            }
        }
    }

    public static void show(char [] num){
        for(int i = 0;i<num.length;i++){
            if(num[i] == '1'){
                System.out.print("A");
            }else
                System.out.print(num[i]);
        }
        System.out.println();
    }

    public static boolean judge(char[] num) { 
        boolean[] bool = new boolean[5]; 
        int size_A = 0, size_2 = 0, size_3 = 0, size_4 = 0; 
        for (int i = 0; i < num.length; i++) { 
            if(num[i] == '1'){size_A++; 
                if(i+2 < num.length && num[i] == num[i+2] || i - 2 > 0 && num[i-2] == num[i]) 
                    bool[0] = true; 
                } 
            if(num[i] == '2'){size_2++; 
                if(i+3 < num.length && num[i] == num[i+3] || i - 3 > 0 && num[i-3] == num[i]) 
                    bool[1] = true; 
                } 
            if(num[i] == '3'){size_3++; 
                if(i+4 < num.length && num[i] == num[i+4] || i - 4 > 0 && num[i-4] == num[i]) 
                    bool[2] = true; 
                } 
            if(num[i] == '4'){size_4++; 
                if(i+5 < num.length && num[i] == num[i+5] || i - 5 > 0 && num[i-5] == num[i]) 
                    bool[3] = true; 
                } 
            } 
        if(size_A == 2 && size_3 == 2 && size_2 == 2 && size_4 == 2){ 
            bool[4] = true; 
        } 
        return bool[0] && bool[1] && bool[2] && bool[3] && bool[4]; 
    } 
}

  2. 分糖果
           有n个小朋友围坐成一圈。老师给每个小朋友随机发偶数个糖果，然后进行下面的游戏：每个小朋友都把自己的糖果分一半给左手边的孩子。一轮分糖后，拥有奇数颗糖的孩子由老师补给1个糖果，从而变成偶数。反复进行这个游戏，直到所有小朋友的糖果数都相同为止。你的任务是预测在已知的初始糖果情形下，老师一共需要补发多少个糖果。
    【格式要求】
    程序首先读入一个整数N(2<N<100)，表示小朋友的人数。接着是一行用空格分开的N个偶数（每个偶数不大于1000，不小于2） 要求程序输出一个整数，表示老师需要补发的糖果数。
    例如：输入
    3
    2 2 4
    程序应该输出：
    4
    资源约定：
    峰值内存消耗（含虚拟机） < 256M
    CPU 消耗 < 1000ms
           解：因为小朋友是做一圈的所以要考虑边界循环回到开头的情况，我的思路是，小朋友的糖果存到一个sugar数组里，然后每个人手里的糖果减半给下一个人，再开一个数组halfsugar，halfsugar[i+1]=sugar[i]/2，然后对sugar数组减半加上halfsugar，就可以得到分糖后，每个小朋友手中有多少个糖果，再由老师补齐，进行下一轮。
    代码如下：

    import java.util.Scanner;

public class B_2014_6 {
    public static int count =0;
    public static void main(String []args){
        Scanner in = new Scanner(System.in);
        int N = in.nextInt();
        int [] sugar= new int[N];
        for(int i = 0;i < N;i++){
            sugar[i] = in.nextInt();
        }
        while(!check(sugar)){
            giveleft(sugar);
            teachergive(sugar);
        }

    System.out.println(count);
}

    public static void giveleft(int []sugar){
        int [] halfsugar = new int[sugar.length];
        for(int i = 0; i < sugar.length;i++){
            if(i+1 != sugar.length)
                halfsugar[i+1] = sugar[i]/2;
            else
                halfsugar[0] = sugar[i]/2;
        }

        for(int i = 0; i < sugar.length;i++){
            sugar[i] = sugar[i]/2 + halfsugar[i];
        }
    }

    public static void teachergive(int []sugar){
        for(int i = 0; i < sugar.length;i++){
            if(sugar[i]%2 == 1){
                sugar[i]++;
                count++;
            }
        }
    }

    public static boolean check(int []sugar){
        int judge = sugar[0];
        for(int i = 0; i < sugar.length;i++){
            if(sugar[i] != judge){
                return false;
            }
        }

        return true;
    }
}
  3. 地宫取宝
           X 国王有一个地宫宝库。是 n x m 个格子的矩阵。每个格子放一件宝贝。每个宝贝贴着价值标签。地宫的入口在左上角，出口在右下角。小明被带到地宫的入口，国王要求他只能向右或向下行走。走过某个格子时，如果那个格子中的宝贝价值比小明手中任意宝贝价值都大，小明就可以拿起它（当然，也可以不拿）。当小明走到出口时，如果他手中的宝贝恰好是k件，则这些宝贝就可以送给小明。请你帮小明算一算，在给定的局面下，他有多少种不同的行动方案能获得这k件宝贝。
    【数据格式】
    输入一行3个整数，用空格分开：n m k (1<=n,m<=50, 1<=k<=12)
    接下来有 n 行数据，每行有 m 个整数 Ci (0<=Ci<=12)代表这个格子上的宝物的价值
    要求输出一个整数，表示正好取k个宝贝的行动方案数。该数字可能很大，输出它对 1000000007 取模的结果。
    例如，输入：
    2 2 2
    1 2
    2 1
    程序应该输出： 2
    再例如，输入：
    2 3 2
    1 2 3
    2 1 5
    程序应该输出： 14
    资源约定： 峰值内存消耗（含虚拟机） < 256M CPU消耗 < 2000ms
           解：小明只能向右或者向下走，根据小学知识，在给定的矩阵中，小明不管选取怎样的路线，走的路程一定是一样的，而每条路线上的宝贝价值又不同，所以才会产生不同的行动方案。但是小明只能拿K件宝贝，也就是说即使选取的同样的路线，也会因为携带的宝贝价值不同而使得行动方案不一样，但是应该考虑的就是当小明选取了不一样的路线拿走了相同价值的宝贝，是否算同一种情况，根据样例输出，应该是算同一种情况的。

社交网站的数据挖掘与分析（一）

这一系列的文章是为了记录我在学习_《社交网站的数据挖掘与分析》第二版 Matthew A. Russell著_ 一书的过程，知识的总结以及思路的梳理，希望通过这本书的学习能够让我初步了解 _社会计算_ 这一领域。

第一章 挖掘Twitter（微博）

       第一章主要围绕着Twitter初步介绍了以后会用到的一些工具及基本知识。
既然是围绕Twitter的，那么，我们首先要知道Twitter是什么呢？根据书中的解释，_Twitter是通过简短对话，将人与其关注的事物相连的的 兴趣图谱（interest graph）以及推文和Twitter的“关注”机制将人们以不同方式联系到一起的社交方式_。有没有觉得和微博很像？
       这里提到了一个概念 兴趣图谱（interest graph ) ，那什么是兴趣图谱呢？兴趣图谱是对人们与他们兴趣之间的关系进行建模的一种方式。兴趣图谱在数据挖掘领域提供了大量的可能性，这些主要涉及测量事物之间的相关性从而进行智能推荐，或涉及机器学习的其他应用。
       Twitter相比之前的社交网站还有一个很重要的特征使得它可以被描述为兴趣图谱。Twitter的标签体系，在介绍这一点时，书中用了一个新的名词 大众分类法，这只是一种描述去中心化标签领域的绝好方式，当允许人们使用标签分类内容的时候，它可以涌现为集体智慧。
       Twitter也是一个 非对称模型，非对称模型很好理解，我关注了范冰冰，而很可能（几乎是一定的…）范冰冰并不会关注我，所以，我和范冰冰之间的关注就不是对称的。
       推文是Twitter的核心，就像我们刷微博是为了看别人的140字，但这140字本身还捆绑了两种需要特别重要的元数据：实体和地点。实体指的是用户的引用，主题标签，URL以及和推文相关的媒体文件（图片，视频），而地点可能写在推文中，比如我在人民广场吃炸鸡，也可能是用户实际的签到位置。
       这是一条实际的微博，原书用的Twitter，我懒得注册，还要翻墙，直接用微博了。
       
       首先是引用了@TECH2IPO创见，然后是微博内容，并给出了两个URL。简短的推文可以蕴含大量信息，可以明确引用多个其他的Twitter用户，到网页的链接，使用主题标签相互引用的主题，而这些话题作为聚集点对整个Twitter虚拟空间（Twitterverse）以一种便于搜索的方式进行垂直切分。
       了解了以上的基础知识之后就可以尝试利用API对数据进行基本的采集和处理工作了，为了方便，书中关于Twitter的分析尽可能用新浪微博来代替。一下部分讲解利用python获取OAuth2.0授权的过程。

蓝桥杯练习（一）

今天没什么事情，上网找了蓝桥杯2014年 _Java b_ 组 的道题做一做，还是挺有意思的…..

  1. 武功秘籍
           小明到X山洞探险，捡到一本有破损的武功秘籍（2000多页！当然是伪造的）。他意识到：书的第10页和第11页在同一张纸上，但第11页和第12页不在同一张纸上。 小明只想练习该书的第81页到第92页的武功，又不想带着整本书。请问他至少要撕下多少张纸带走？
           解： 这是一道送分题，问题是需要拿走多少页。很显然的思想是把起始页和末尾页相减再除2加1，唯一的干扰是，根据题目的信息，书是从偶数页开始的，那么，如果起始页是奇数，将其减1，才能得到正确的页数。
    代码如下：

    package cmy;

import java.util.Scanner;
public class B_2014_1 {
    public static void main(String[] args) {
    // TODO Auto-generated method stub
    Scanner in = new Scanner(System.in);
    int start = in.nextInt();
    int end = in.nextInt();
    int sum;
    while(!(start % 2 == 0)){
        if (start % 2 == 1){
            start = start - 1;
        }
    }
    sum = (end - start) / 2 + 1;
    System.out.println(sum);
    in.close();
    }
}

  2. 切面条
           一根高筋拉面，中间切一刀，可以得到2根面条。如果先对折1
    次，中间切一刀，可以得到3根面条。如果连续对折2次，中间切一刀，可以得到5根面条。那么，连续对折10次，中间切一刀，会得到多少面条呢？
           解：一道找规律的题，根据观察写出数列，（0，2），（1，3），（2，5），不难得到通项公式$$a_n={2^n+1}$$根据通项公式，就能够很容易的解出答案了。
    代码如下：

  3. 猜字母
           把 a，b，c，d，…，s共19个字母组成的序列重复拼接106次，得到长度为2014的串。接下来删除第1个字母（即开头的字母a），以及第3个，第5个等所有奇数位置的字母。得到的新串再进行删除奇数位置字母的动作。如此下去，最后只剩下一个字母，请写出该字母。
           解：很简单，先用串A保存拼接后长度为2014的字符串，因为对其进行删除奇数位的操作，保留的就是偶数位，那么就用串B把A中的偶数位保留下来，把A的内容清空以后，把B的内容赋值给A，清空B再重复操作，直到B中只剩一个字母。
    代码如下：

    package cmy;

import java.util.Scanner;
public class B_2014_3 {
    public static void main(String[] args) {
        // TODO Auto-generated method stub
        //Scanner in = new Scanner(System.in);
        StringBuilder A = new StringBuilder();
        StringBuilder B = new StringBuilder();
        for (int i = 0; i < 106; i++){
            for (int j = 0; j < 19; j++){
                A.append(String.format("%c", ('a' + j)));
            }
        }

        while(A.length() != 1){
            for(int i = 0;i < A.length(); i++){
                if((i + 1) % 2 == 0){
                    B.append(A.charAt(i));
                }
            }
            A.delete(0, A.length());
            A.append(B);
            B.delete(0, B.length());
        }

        System.out.println(A.toString());
    }
}

  6. 奇怪的分式
           上小学的时候，小明经常自己发明新算法。一次，老师出的题目是：
    1/4乘以 8/5 小明居然把分子拼接在一起，分母拼接在一起，答案是：18/45 。老师刚想批评他，转念一想，这个答案凑巧也对啊，真是见鬼！对于分子、分母都是 1~9 中的一位数的情况，还有哪些算式可以这样计算呢？请写出所有不同算式的个数（包括题中举例的）。 显然，交换分子分母后，例如：4/1 乘以 5/8 是满足要求的，这算做不同的算式。但对于分子分母相同的情况，2/2 乘以 3/3 这样的类型太多了，不在计数之列!
    注意：答案是个整数（考虑对称性，肯定是偶数）。
           解：没找到什么规律，所以直接暴力找了。
    代码如下：

    public class B_2014_6 {
    public static void main(String[] args) {
        // TODO Auto-generated method stub
        int a,b,c,d;
        int x,y;
        int count = 0;
        for(int i = 1;i <= 9; i++){
            a = i;
            for(int j = 1;j <= 9; j++){
                b = j;
                if(a == b)
                    continue;
                else{
                    for(int k = 1;k <= 9; k++){
                        c = k;
                        for(int n = 1;n <= 9; n++){
                            d = n;
                            if(c == d)
                                continue;
                            else{
                                x = 10*a+c;
                                y = 10*b+d;
                                if((double)x/y == (double)(a*c)/(b*d)){
                                    System.out.println(String.format("%d/%d,%d/%d", a,b,c,d));
                                    count++;}
                                else
                                    continue;
                            }}}}}}
        System.out.println("Count:" + count);
    }
}
